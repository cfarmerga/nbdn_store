Request Handling

RawHandler [Front Controller]

  • Interface with ASP.Net mechanics
  • Forward control to our front controller mechanism to process the actual work

FrontController [ Gateway ]

  • Finds and invokes the correct application behaviour using a registry of all the behaviours that are able to be performed.

CommandRegistry [ Registry ]

  • Retrieves the correct command that can process the actual request

DefaultRequestCommand [ Specification ],[ Visitor ]

  • Determines if it can process a request and forwards control to the application specific behaviour if it can.

Request Processing

ApplicationCommand [ Strategy ]

  • There will be an implementation of this for each application specific behaviour

Response Processing

ResponseEngine [ Strategy ]

  • Render the view associated with the corresponding display model

ViewRegistry [ Registry ]

  • Find the view associated with the display model

ViewPathRegistry [ Registry ]

  • Find the view path associated with the display model
